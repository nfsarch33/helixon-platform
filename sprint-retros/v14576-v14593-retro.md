## What worked

1. **Hash-only audit logging**: SHA-256 fingerprints of every secret value captured; no plaintext in `evidence/`. Strict no-shell-leak maintained across all 18 sprints.
2. **Vendor verification**: every binary, image, Helm chart, MCP server checked against upstream. Documented in evidence.
3. **Idempotency**: re-running scripts (svcregistry-bridge, secrets-bootstrap, fleet-sync-ssh-config) is safe.
4. **Operator-friendly carry-forwards**: every CF has owner + severity + sprint target. No mystery items.
5. **Browser automation SOP**: agent-browser (Vercel Labs) adopted as the documented path for OCI web UI tasks (operator directive 2026-07-10).
6. **Production-ready ADR-0098**: canonical win1/wsl1 topology captured with load-bearing vs informational classification.
7. **WSL password migration strategy**: 8-phase SOP with safe pilot-on-win4-first approach.

## What didn't work

1. **Rootless Podman + GitLab CE**: UID-mapping issues for the complex multi-service GitLab CE container. Pivoted to gitlab-runner-only deployment.
2. **SprintBoard API health probe hangs**: blocks live ticket creation. Workaround: NDJSON export.
3. **Operator presence required** for actual secret rotation: cannot push keys to vendor consoles from wsl1.
4. **`oci` CLI not installed**: blocks OCI automation from wsl1.
5. **Tailscale auth keys expired**: replacement requires Tailscale admin console + 1Password update; deferred to operator.
6. **OCI jumps (oci-sydney-jump, oracle-jump)**: both unreachable for different reasons (offline 64d / SSH key rejected). Recovery requires operator MFA.
