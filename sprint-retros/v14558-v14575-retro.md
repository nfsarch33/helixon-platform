## Why this retro is concise

This was a **carry-forward execution** arc, not a greenfield one. Most
of the architectural decisions were captured in ADR-0095 (service
registry) and ADR-0096 (win4 recovery). The only new architecture in
this arc is:

1. **`fleet-agent` boot-time self-registration** (canonical schema)
   — formalized into `~/.local/bin/fleet-agent-register.sh` to
   insulate the systemd unit from `$$VAR` interpolation bugs.
2. **`fleet-registrar` on wsl1** — second instance of the central
   registrar pattern, decoupled from the legacy `jason@wsl2` user.
3. **1Password rotation gating** — formalized into a dedicated runbook
   with hard-exclusion table.

See **ADR-0097** (`docs/adrs/ADR-0097-service-registry-live-and-bridged.md`)
for the canonical reference.

## Risks / Watch-list

- `wsl1 → win1` is forbidden (wsl1 uses WSL2 mirror network); all SSH
  automation uses Tailscale. If Tailscale is down, fleet-wide SSH is
  unavailable.
- `k3s` on wsl1 is single-control-plane; HA requires moving control to
  win2 or k3s-embedded etcd (a v14577+ arc).
- Grafana + Prometheus use rootless Podman containers; if a Windows
  Update resets WSL, the containers are recreated on next boot via
  `enable-linger` + podman auto-restart.
