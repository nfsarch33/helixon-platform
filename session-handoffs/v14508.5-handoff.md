### Phase 0.2 — op cli write-path fix

**Root cause**: `op` 2.34.1 requires the 1Password desktop app to
mediate write operations. With debug logging
(`OP_DEBUG=1`), the CLI explicitly logs:

> Skipped loading desktop app settings file. The desktop app might
> not be installed: read file: lstat
> /home/jaslian/.config/1Password/settings/settings.json: no such
> file or directory

…and then waits forever for the desktop IPC channel.

**Fix path (operator action, not skippable)**: install 1Password
desktop app on win1, then in 1Password > Developer > Settings enable
"Integrate with the 1Password SDKs" / "Integrate with other apps".

**Fallback path (used this sprint)**: a Go CLI bootstrap
`cmd/onepassword-bootstrap` that uses the **official 1Password Go SDK**
(`github.com/1password/onepassword-sdk-go v0.4.0`, MIT, 1Password org,
pkg.go.dev canonical) to authenticate with the service account token
and call `client.Items().Create(ctx, params)` directly. The SDK ships
WASM auth logic so it does not require the desktop app.

**TDD test**: `tests/test_op_cli_write.py` (6 tests, all PASS):

1. test_01_whoami_returns_service_account — op whoami reports SERVICE_ACCOUNT
2. test_02_vault_list_contains_cursor_ironclaw — vault is visible
3. test_03_read_existing_login_password — op read of the existing item
4. test_04_item_create_hangs_without_desktop_app — regression guard
5. test_05_bootstrap_binary_builds — go build produces the binary
6. test_06_bootstrap_idempotent — re-run skips existing items
