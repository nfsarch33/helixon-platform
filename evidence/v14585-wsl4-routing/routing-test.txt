=== Background ===
Pair 5 MVP (v14584) installed Qwen3.6-27B GGUF on wsl4 (cf-v14584-01 carried since v14530).
However, wsl4 RTX 3090 (24GB VRAM) was already running vllm serving
Qwen/Qwen2.5-7B-Instruct-AWQ on port 8001 — a 7B AWQ model that fits in
the 24GB VRAM while a 27B Q8_0 (29GB) does not. Per the operator's CF-v14584-01
resolution, we wired the existing vllm 7B AWQ cell as c5-wsl4 (tier 1 fallback)
and re-tasked c7-wsl1 (dual RTX 3090 + llamacpp port 8010) to the primary
Qwen3.6-27B-MTP-Q8_0 (29GB).

=== Routing schema discovered ===
1. node.cfg.Tier is matched EXACTLY against the X-Tier HTTP header (string equality).
   Range strings like '0..3' do NOT match any tier.
2. node.cfg.Models is matched EXACTLY against the 'model' field in the request body.
3. Routes block in YAML is parsed but NOT applied by current binary build.
4. Each node is probed on /health, with /v1/models fallback on 404 (probe.go).
5. A node with /v1 suffix in URL breaks health probe (path concatenation bug).
