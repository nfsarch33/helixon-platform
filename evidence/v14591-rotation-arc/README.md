## Retro: what worked / what didn't

### What worked

1. **Hash-only audit logging** (SHA-256 of current value, no plaintext in evidence/) — strict no-shell-leak maintained.
2. **Hard exclusion list** — explicit policy preventing rotation of items that would lock us out.
3. **Vendor URL mapping** — every rotatable item has a clear path to rotate (no guessing).
4. **SprintBoard NDJSON export** — works around SprintBoard API hang (CF-v14590-01).
5. **WSL pilot-on-win4-first strategy** — safe, easy rollback.

### What didn't work

1. **SprintBoard API health probe hangs** (`curl /healthz` → no response) — blocked live ticket creation. Workaround: NDJSON export for later import.
2. **Operator presence required** for actual key rotation — we can only inventory + plan, not execute.
3. **Tailscale auth key replacement** is operator-dependent (admin console login).
4. **OCID `oci` CLI not installed** on wsl1 — blocked OCI automation (CF-v14588-04 still open).

### Recommendations for next arc

1. **Fix SprintBoard API health probe hang** (CF-v14590-01). Likely cause: SQLite db lock or stale migration.
2. **Install `oci` CLI on wsl1** (CF-v14588-04). Then re-key the OCI API from a fresh auth token.
3. **Trigger WSL password migration** (CF-v14591-03) when all 7 pre-requisites met.
4. **Add calendar reminders** for next Tailscale auth key rotation (2026-10-08, 90 days from replacement).
