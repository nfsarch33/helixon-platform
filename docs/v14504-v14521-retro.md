## 7. Lessons learned

1. **Pair-lock introduced late (v14516)** — should have been part of
   v14504. Cost us two hours of rebasing PRs #7 and #8 in Phase 0.
2. **`op cli write-path` is fragile** — desktop-app dependency makes
   `op item create` unreliable. The 1Password Go SDK fallback (v14508.5)
   is now the canonical path.
3. **Conventional-commit hooks** — every test that creates a real git
   repo needs `chore: ...` style messages. Catch this in CI earlier.
4. **Branch protection + first push** — newly-created GitHub repos
   block direct `main` push; use a feature branch + PR. The
   `gh pr create` API is reliable but takes 2-3 round trips.
5. **PowerShell bash heredoc** — large multi-line scripts need to be
   staged to /mnt/c/.../Temp/ first, then executed. Direct wsl -e
   invocations often fail.
6. **EvoSpine cycle driver works** — first run committed
   `aa8eaa4`; the obs→hyp→eval→commit pipeline is real.

## 8. Recommendations for the next roadmap (v14522+)
